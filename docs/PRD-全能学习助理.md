# PRD：全能学习助理

## 1. Problem Statement

AI 应用开发实习项目不能只停留在“接一个大模型聊天窗口”。对 HR 和技术面试官来说，一个有价值的项目需要证明候选人能围绕真实学习场景完成产品设计、前端高保真交互、后端 API、知识库构建、RAG 检索、Agent 编排和工程化交付。

学生学习与求职准备天然包含资料导入、知识问答、复习记忆、模拟面试、学习规划等任务，适合作为 AI 应用项目载体。

## 2. Product Positioning

**全能学习助理** 是一个面向学生的 AI 学习知识工作台，而不是单一聊天机器人。

产品提供三种核心模式：

1. **Chat 模式**：普通 AI 对话，用于快速问答、解释概念、验证模型调用。
2. **RAG 模式**：围绕用户导入的知识库进行资料问答，回答带来源片段。
3. **Agent 模式**：面向学习任务的角色化助手，可选择教育专家、面试官、出题教练、总结教练等 SubAgent。

正式产品采用前后端分离：

- 前端用 Vue 3 高保真还原 `prototype/index.html`。
- 后端用 FastAPI 承载 LLM、RAG、Agent、存储和流式输出。
- LangChain / LangGraph 作为后端 AI 编排能力逐步接入。

## 3. Target Users

- 学生：需要课程答疑、资料总结、复习记忆和面试准备。
- 求职者：需要围绕简历项目进行模拟面试和技术追问。
- 项目作者：需要一个可演示、可复现、可写进简历的 AI 应用项目。
- 面试官：需要通过项目判断候选人的 AI 应用工程能力。

## 4. Product References

本项目参考 NotebookLM 和 ima 的产品方向，但不做商业级完整复刻。

- NotebookLM 启发：围绕用户导入资料进行问答、总结、学习材料生成和来源引用。
- ima 启发：用清晰入口组织 AI 知识工作台，如新对话、知识库、问答历史。
- 本项目取舍：Chat 保持纯会话体验，RAG 聚焦知识库问答，Agent 聚焦学习角色和多步骤学习任务编排。

## 5. Core User Stories

### 5.1 Chat

1. 作为学生，我想直接和 AI 对话，以便快速解释一个概念。
2. 作为学生，我想获得流式回复，以便交互更自然。
3. 作为用户，我想控制是否开启联网搜索，以便在需要实时信息时增强回答。
4. 作为用户，我想选择思考深度，以便在速度和推理质量之间取舍。
5. 作为开发者，我想通过 Chat 模式验证前端 SSE 和后端 DeepSeek 调用层。

### 5.2 Knowledge Base

1. 作为学生，我想导入 Markdown、TXT、PDF 等资料，以便建立自己的学习知识库。
2. 作为学生，我想导入在线链接，以便把网页资料纳入学习范围。
3. 作为学生，我想看到文档切分后的片段预览，以便确认知识库质量。
4. 作为学生，我想按知识库管理资料，以便区分不同课程、项目和面试主题。
5. 作为开发者，我想记录文档来源、chunk 数量和更新时间，以便调试 RAG 效果。

### 5.3 RAG

1. 作为学生，我想基于指定知识库提问，以便获得更贴合资料的回答。
2. 作为学生，我想看到回答引用的来源片段，以便判断答案可信度。
3. 作为学生，我想在回答不充分时继续追问，以便深入理解。
4. 作为用户，我想调整 top_k、阈值、reranker 和回答风格。

### 5.4 QA Cards

1. 作为学生，我想把一次问答保存为问答卡片，以便后续复习。
2. 作为学生，我想从知识库自动生成问答对，以便快速构建复习材料。
3. 作为学生，我想通过翻牌方式查看问题和答案，以便进行主动回忆。
4. 作为学生，我想给卡片标记掌握程度，以便知道哪些知识点需要复习。

### 5.5 Agent

1. 作为求职者，我想调用面试官 SubAgent 根据我的项目进行追问，以便准备技术面试。
2. 作为学生，我想调用教育专家 SubAgent 帮我制定学习路线，以便合理安排学习。
3. 作为学生，我想调用出题/总结类 Skills，以便完成练习生成和资料总结。
4. 作为学生，我想调用出题教练 SubAgent 结合知识库生成练习题，以便检验掌握程度。
5. 作为学生，我想调用总结教练 SubAgent 把资料整理成提纲和复习要点，以便高效复盘。

## 6. Functional Requirements

### 6.1 Global Layout

- 页面顶端提供三模式快捷切换：`Chat`、`RAG`、`Agent`。
- `Knowledge`、`Cards/History` 作为侧边栏或二级入口，不与三种推理模式混淆。
- 模式切换后，页面标题、设置栏、输入区、右侧 Inspector 和结果展示全部切换。
- 前端一比一还原 `prototype/index.html` 的主布局、Neural Expressive 视觉语言和核心交互。

### 6.2 Chat Mode

- 支持普通聊天。
- 支持 SSE 流式输出。
- 支持保存会话历史。
- 不依赖知识库。
- 不支持 Skills、MCP、SubAgent、知识库选择。
- 支持联网搜索开关、思考深度、模型选择、温度、上下文保留开关。
- Chat 页面右侧 Inspector 显示会话信息、联网搜索状态和引用结果，不显示知识库参数。

### 6.3 Knowledge Mode

- 支持创建、选择、删除知识库。
- 支持导入 Markdown / TXT；PDF 作为简历版目标；URL 作为增强目标。
- 支持 chunk size、chunk overlap、标题/段落切分策略。
- 支持切分预览、导入状态、失败原因和重建索引入口。

### 6.4 RAG Mode

- 支持选择知识库后提问。
- 检索相关 chunks。
- 基于 chunks 生成回答。
- 展示来源片段、文档名、chunk 序号、相似度分数。
- 支持无检索结果兜底。
- 不支持通用 Skills、MCP、SubAgent。
- 支持保存问答卡片。

### 6.5 QA Cards

- 支持手动保存问答为卡片。
- 支持从知识库或 RAG 结果生成问答卡片。
- 支持翻牌查看答案。
- 支持标记“未掌握 / 模糊 / 已掌握”。
- 支持按知识库、标签、掌握程度筛选。

### 6.6 Agent Capability Layer

能力层只在 Agent 模式面向用户开放，Chat/RAG 不直接调用 Skills/MCP/SubAgent。当前用户侧重点是可选择的学习角色；MCP 作为后端扩展能力预留，不作为本阶段核心入口。

- **Skills**：轻量、可复用、单步能力，例如总结、改写、出题、生成问答卡片。
- **MCP Tools**：协议化外部工具，例如网页读取、知识库检索、文件解析，后续按需接入。
- **SubAgent**：带角色、目标和策略的专门助手，例如教育专家、面试官、出题教练、总结教练。

### 6.7 Agent Mode

- 支持任务拆解。
- 支持状态跟踪。
- 支持多步骤工具调用。
- 支持调用一个或多个 SubAgent。
- 支持最终结果汇总。
- 支持学习规划、模拟面试、出题练习、资料总结等任务。
- 支持教育专家、面试官、出题教练、总结教练等角色模式。
- 不提供 cowork、云端/本地工作场景入口。

## 7. Non-functional Requirements

- 高保真：正式前端应尽量一比一还原原型视觉与交互。
- 本地优先：不依赖生产部署即可演示。
- 密钥安全：API Key 不入库，前端不持有模型 Key。
- 可复现：README 必须写清安装、配置、启动和演示步骤。
- 可降级：PDF/URL 解析失败时不能影响 Chat 模式。
- 可解释：RAG 回答展示来源片段。
- 可测试：优先测试文档切分、路由、卡片存储、模式边界和 API schema。

## 8. Implementation Decisions

- 前端：Vue 3 + Vite + TypeScript + Tailwind CSS + Pinia + Vue Router。
- 前端组件：Composition API + `<script setup lang="ts">`，组件小而聚焦，props down / events up。
- 后端：FastAPI + Python 服务层。
- 模型：DeepSeek OpenAI-compatible API。
- AI 编排：LangChain（TextSplitter、Embeddings、Chat Model、RetrievalQA）+ LangGraph（Agent 编排）+ Chroma 向量库。
- 向量库：Chroma。
- 元数据存储：SQLite。
- 流式输出：后端 SSE，前端 EventSource 或 fetch stream。
- Streamlit：不再作为正式 UI 主线；旧 UI 代码已从新主线清空。

## 9. Acceptance Criteria

### MVP

- Vue 前端能一比一还原原型主界面轮廓。
- FastAPI 后端提供 Chat API。
- Chat 模式可真实调用模型并流式输出。
- 可以创建知识库。
- 可以导入 Markdown/TXT。
- 可以完成文档切分和预览。
- RAG 模式可以基于知识库回答。
- 可以手动保存问答卡片并翻牌。

### 简历版

- 支持 PDF 导入。
- 支持 URL 导入。
- Chat 支持联网搜索开关和思考深度选择。
- RAG 支持知识库选择、检索参数调整和来源片段展示。
- Agent 支持学习角色 SubAgent、学习 Skills、任务拆解和结果汇总。
- 支持 SQLite 持久化。
- 有 README、架构图、演示截图和典型演示脚本。

### 增强版

- 支持 MCP 工具化。
- 支持 Reranker。
- 支持 LangSmith 或等价可观测。
- 支持 Docker 化。
- 支持更多文档格式和更完整的卡片复习策略。

## 10. Out of Scope

- 不做用户账号系统。
- 不做多人协作。
- 不做生产级权限管理。
- 不做模型训练或微调。
- 不做复杂在线部署，除非 MVP 和简历版已完成。
