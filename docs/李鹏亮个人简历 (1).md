### AI Learning Assistant

_独立开发 | 2026/05 - 至今_

面向学生学习和求职准备场景的 AI 应用项目，聚焦普通 Chat、知识库 RAG 问答、问答卡片复习，以及教育专家、面试官等学习角色助手。

项目地址：[**github.com/lplnb-del/ai-learning-assistant**](https://github.com/lplnb-del/ai-learning-assistant)

技术栈：Vue3 + Vite + TypeScript + Tailwind CSS + Pinia + Vue Router + FastAPI + Python + SQLite + DeepSeek Gateway + RAG + pytest

负责模块与个人贡献：

-   **前端工作台与模式边界**：负责 Vue3 + TypeScript 工作台页面搭建，拆分 Chat、RAG、Agent、Inspector、知识库和问答卡片等视图；通过 Pinia、Vue Router 和组件化拆分维护页面状态与模式切换

-   **后端接口与模型网关**：负责 FastAPI 基础接口、统一错误响应、CORS 和 OpenAPI 配置；封装 DeepSeek OpenAI-compatible Gateway，实现普通 Chat 与 SSE 流式输出，确保 API Key 只保存在后端环境变量中

-   **知识库导入处理**：负责知识库创建、列表、删除、Markdown/TXT/PDF/URL 导入、文本清洗切分、chunk 预览和索引重建接口，使用 SQLite 保存知识库、文档和 chunk 元数据

-   **RAG 问答链路**：负责本地 hashing embedding、JSON 持久化向量索引和 cosine similarity 检索，实现基于知识库的问答接口，返回来源文档、chunk 序号、检索分数和 prompt 预览，便于解释答案依据

-   **工程化与文档**：补充模型无关后端测试，覆盖知识库导入、索引重建和 RAG 问答链路；维护 README、开发计划和验收计划，当前后端 35 个 pytest 用例通过，前端 lint、typecheck 和 build 均通过

后续计划：接入 Chroma、问答卡片和角色化 Agent Capability Layer，优先完善教育专家、面试官、出题教练、总结教练等可选学习角色
